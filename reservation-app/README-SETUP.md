# Philosoffee Reservation App Setup Guide

I have successfully created a complete Next.js reservation web application that perfectly matches your premium design aesthetics. 

The application is located in the `reservation-app` folder. It includes the frontend forms, and a serverless API route ready to connect to a free-tier database (Supabase) and WhatsApp API (Twilio).

## How to Run Locally

1. Open a new terminal and navigate to the new app directory:
   ```bash
   cd reservation-app
   ```
2. Start the development server:
   ```bash
   npm run dev
   ```
3. Open your browser and go to `http://localhost:3000` to see the reservation form.

## Setting up the Backend (Free Tier Services)

To make the database and WhatsApp integration work, you need to set up free accounts on Supabase and Twilio, and then add your API keys.

### 1. Database (Supabase)
Supabase provides a powerful, free-tier Postgres database perfectly suited for this.
- Go to [Supabase](https://supabase.com) and create a free project.
- Once created, go to the **SQL Editor** and run this query to create your table:
  ```sql
  create table reservations (
    id bigint generated by default as identity primary key,
    created_at timestamp with time zone default timezone('utc'::text, now()) not null,
    date text,
    time text,
    guests text,
    name text,
    email text,
    phone text
  );
  ```
- Go to **Project Settings -> API** to get your `Project URL` and `service_role` secret key.

### 2. WhatsApp Messages (Twilio)
Twilio offers a free Sandbox for sending WhatsApp messages.
- Go to [Twilio](https://twilio.com) and create a free account.
- Activate the **Twilio Sandbox for WhatsApp** (it will give you a phone number to test with).
- Grab your `Account SID` and `Auth Token` from the Twilio console.

### 3. Add Environment Variables
Inside the `reservation-app` folder, create a file named `.env.local` and add your keys:

```env
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key

TWILIO_ACCOUNT_SID=your_twilio_account_sid
TWILIO_AUTH_TOKEN=your_twilio_auth_token
TWILIO_WHATSAPP_NUMBER=whatsapp:+14155238886
OWNER_WHATSAPP_NUMBER=whatsapp:+919876543210
```
*(Replace the values with your actual keys and the cafe owner's phone number).*

## Deployment

Because this app requires a serverless backend for the API route (to securely hide your API keys), it cannot be hosted on GitHub Pages like the static HTML site. 

The best (and free) way to deploy this is using **Vercel**:
1. Push this `reservation-app` folder to your GitHub repository.
2. Go to [Vercel](https://vercel.com) and import your repository.
3. Select `reservation-app` as the Root Directory.
4. Add the Environment Variables from your `.env.local` file into Vercel's Environment Variables section.
5. Click **Deploy**.

Once deployed, copy the Vercel URL (e.g., `https://philosoffee-reserve.vercel.app`) and update the "CONFIRM RESERVATION" button link in your main `Philosoffee.html` to point to this new app!
